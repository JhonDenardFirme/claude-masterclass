Module Introduction

Volume 3 covers capability depth and competitive context: what Opus 4.7 and Claude 4.1 actually do differently at the mechanism level, and where Claude sits relative to ChatGPT 5.3/5.4 on a task-by-task basis. Self-verification is a workflow shift, X-High is a surgical tool, and the ChatGPT comparison is a task-specific decision framework — not brand loyalty.

A A. Claude Opus 4.7 — Four Capabilities Overview A
A1Landscape Position

Claude Opus 4.7 is Anthropic's current flagship — a targeted upgrade, not an across-the-board step change. Exclusive to Opus 4.7: self-verification · X-High thinking · 2,576px HD vision · literal instruction following. Sonnet users see incremental gains only. Access: Pro or Team on claude.ai; API at ~15% more per output token than prior Opus, with higher per-response token consumption due to the verification pass. Benchmark: trading punches with GPT-5.4 on coding, ahead of Gemini 3 Ultra on vision, roughly tied on raw reasoning.

›Four exclusive features. ~15% cost increase. Targeted upgrade, not universal improvement.
A2Four Features at a Glance

Self-verification: internal review pass that catches errors before streaming. X-High thinking: up to 30,000 thinking tokens — expert-depth analysis tier. HD vision: 1,092px → 2,576px (~6× pixel count) — dense dashboards and fine text now readable. Literal instruction following: reweighted compliance vs. helpfulness — what you ask is what you get, no added extras.

›Each feature targets a specific prior failure mode. Not decoration — each solves a documented problem.
B B. Self-Verification Deep Dive B
B1How It Works

On prompts involving code, logic, math, or multi-step reasoning: Claude generates a response internally, runs a verification pass (checking consistency, edge cases, prompt compliance), revises if issues are found, then streams the post-verification version. The user only ever sees the corrected output. Simple queries (factual, conversational) don't trigger it — no speed penalty on those. Trigger cost: 20–40% longer response time on substantive prompts.

›Fires on: code · logic · math · multi-step reasoning. No penalty on: facts, chat, basic writing.
B2The SQL Test

Same prompt: SQL query for weekly active users handling mid-week signups, soft-deleted users, and global time zones. Claude 4.6: ~8 seconds, looked clean, silently dropped non-UTC users on week boundaries — 12% error, undetected. Claude 4.7: ~22 seconds, flagged the UTC boundary issue explicitly, provided the corrected query converting to user local time first — correct output. Source framing: 4.6 is confident and wrong. 4.7 is slower and right.

›The 12% error would have driven wrong decisions in dashboards. Self-verification caught it before shipping.
B3The Code Review Test

Code review across 20 TypeScript files. Claude 4.6: 7 real bugs found, 3 hallucinated false positives. Claude 4.7: 11 real bugs found, 0 false positives. Each false positive = 10 minutes chasing a non-existent problem. Zero hallucinations means code review output can be trusted and acted on directly — the secondary verification step that was previously necessary with 4.6 is eliminated.

›Eliminating false positives is more valuable than finding extra real bugs in review workflows.
B4The Trust-Model Shift

Self-verification moves Claude from "useful assistant I verify" to "competent collaborator I spot-check." Source's one-line: "The first model where I trust the first draft on serious work." Correct strategy: use 4.7 for the hard tasks where the 20–40% latency penalty is also the accuracy dividend. Don't avoid 4.7 to skip the wait — that wait is the error-catching pass.

›The latency cost and the accuracy gain are the same thing. Use 4.7 precisely where speed matters less than correctness.
C C. X-High Thinking & HD Vision C
C1Three-Tier Thinking Ladder

Standard: hundreds of thinking tokens, fast, everyday tasks. High: thousands of thinking tokens, prior maximum, moderately complex. X-High: up to ~30,000 thinking tokens, new ceiling, expert-depth analysis. Total token cost at X-High: ~40,000 tokens per response (thinking + output). Time: ~90 seconds for expert-level complex tasks.

›X-High is the only tier that finds what a human expert finds in an hour of deep reading.
C2Security Audit Test

Mid-sized fintech architecture (6 microservices, shared auth, 2 databases, external payment provider). Issues found by mode: Standard: 6 (surface — missing rate limiting, PII logging, CVE-flagged SDK). High: 9 (more nuanced). X-High: 14 — including a trust boundary violation requiring chaining three separate components to exploit. The jump from High to X-High (9→14) was larger than Standard to High (6→9). The X-High issues were the hardest to find.

›X-High doesn't find more of the same issues — it finds qualitatively different, deeper issues.
C3Surgical Deployment Rule

Deployment rule: use X-High for tasks where you'd otherwise pay an expert hourly — legal analysis, security reviews, financial modeling, deep code audits. Rate limit impact: source went from hitting Pro limits once per week on 4.6 to three times in a single test day on 4.7 when leaning on X-High. Never use for 5-minute tasks — the marginal gain doesn't cover the cost.

›X-High is surgical, not default. Reserve it. Monitor rate limits actively.
C4HD Vision Upgrade

Resolution: 1,092px (4.6) → 2,576px (4.7). ~2.4× linear resolution, ~6× pixel count. Now reliably readable: Bloomberg-style financial dashboards, dense tables, handwritten wireframes, medical results. Source's test: 2,400×600px financial screenshot — 4.6 could read but not analyze; 4.7 answered specific panel queries accurately. New workflow: screenshot any dashboard → ask Claude to extract structured data → trust the output. PDF limitation: most integrations still downscale PDF pages. Workaround: screenshot the page and upload the image directly.

›PDF workaround: screenshot the page, upload as image. Don't upload the PDF file.
C5Literal Instruction Following

Claude 4.6 problem: quietly added extras (a sixth data point, framing paragraphs, inferred features) despite explicit constraints. Claude 4.7 fix: reweighted compliance over helpfulness. Test: 150-word product description, 5 specific facts, zero adjectives, no unmentioned features. 4.6 added extras. 4.7 returned exactly 5 facts, 0 adjectives, nothing invented. Best for: legal/compliance writing (every added word is liability) · structured data extraction (exact schema returned) · prompt chaining (fixed output format for downstream systems). Source's result: eliminated the post-processing cleanup step entirely — ~40 min/week saved.

›Literal instruction following is a calibration change, not a new feature. The model now respects constraints as hard limits.
D D. Claude 4.1 — Core Features D
D1Artifacts

Artifacts — a side panel where Claude's finished work renders as an actual interactive output: code that executes in Claude's sandbox, HTML/CSS/JS that previews live, formatted documents, interactive apps. Prior Claude = copy code and run it yourself. Claude 4.1 = runs code, shows executed output, debugs, shows new output — all inside the artifact panel. Iterative refinement: click any element → tell Claude what to fix → only that element updates. Create manually: Artifacts tab → New Artifact → blank canvas.

›Mindset shift: stop getting text to copy, start building finished interactive outputs.
D2Projects

Projects = persistent workspaces with 200K tokens of context maintained permanently. Upload documents once; every chat inside the project inherits full context. Solves prior Claude's context-drop on new chat. Connects to Google Drive — point Claude at a folder, treated as project knowledge. After ~12 hours of use, Claude generates project-specific memories that enhance workflows further.

›Projects are the solution to "re-upload everything in every new chat." Set up once, context persists forever.
D3Extended Thinking — Live Mid-Process Steering

Toggle below the prompt bar before submitting. Makes reasoning visible step-by-step. Key new capability: live mid-process correction — if Claude takes a wrong turn at step 4, say "check that assumption" and Claude reroutes immediately rather than completing a wrong path. Prior failure mode: see wrong answer, backtrack. New mode: catch wrong turn, steer, continue. Time: 30 seconds–several minutes. Use for: debugging, data analysis, nuanced essays — not simple factual queries.

›The real upgrade isn't seeing the thinking — it's being able to redirect mid-process without restart.
D4Built-in Data Analysis

Drag CSV into chat → plain-English request → Claude writes Python, executes it, plots using Matplotlib, and interprets the results — all inside the artifact panel. No Excel, no Jupyter, no coding knowledge required. Key advance: interprets visuals, not just generates them. Multi-source cross-reference: compare a report and a spreadsheet in one prompt, get summary plus flagged discrepancies without a second prompt.

›Claude acts as a personal data scientist. You describe the analysis in English; Claude handles the code.
D5Custom Styles

Five built-in presets: Normal · Concise · Explanatory · Learning · Formal. Custom styles created two ways: (1) describe tone in a prompt → save as named style; (2) upload writing samples → Claude extracts and saves your voice. Sample quality = style quality: large, consistent samples → strong voice; mixed tones → averaged output. Use case: one style for social media, another for clients, another for technical docs — one-click switching.

›Upload your own writing as samples. One-click switching between context-appropriate voices.
E E. Migrating from ChatGPT E
E1The Privacy Case

OpenAI removed their military-use ban and announced a Pentagon cybersecurity partnership. Anthropic declined the same deal. Source's framing: privacy concern is the trigger; performance advantage is the actual case. Anthropic founded by former OpenAI researchers who left over direction disagreements. Constitutional AI framework: helpfulness, harmlessness, honesty — not helpfulness to defense contractors. Enterprise trust factor: regulated industries (healthcare, finance, legal) cannot use AI built by a defense contractor.

›Even if the Pentagon connection doesn't concern you — why choose the inferior tool?
E2Memory Import Workflow

Claude Settings → Capabilities → Import memory from other AI providers → Start Import → Claude generates extraction prompt. Pro tip: before pasting in ChatGPT, enable all three personalization options (saved memories · browser memories · chat history), then switch ChatGPT model to Thinking. Thinking model runs more search cycles — source reports 3× more memories extracted vs. default model. Copy the output code block → return to Claude → paste → Add to Memory. Claude categorizes and auto-updates every 24 hours.

›The Thinking model switch is the critical step. Default model misses most memories.
E3Full Export & Custom GPT Migration

Full export: ChatGPT Settings → Data Controls → Export Data → .zip with all conversations. Pre-migration trick: ask ChatGPT to analyze your history and generate a working preferences guide → paste into Claude custom instructions. Custom GPT migration: open GPT → Edit GPT → copy Instructions → create Claude Project → paste into Project custom instructions. Deletion: 30 days to finalize; some data may persist per OpenAI's privacy policy.

›Style guide trick turns your ChatGPT history into a working manual for Claude automatically.
F F. Claude vs ChatGPT 5.3 / 5.4 — Head-to-Head F
F1ChatGPT 5.3 — Three Failures

ChatGPT 5.3 ("Project Garlic"): stripped-down, direct, no preambles. Three documented failures. 1. Creativity: "10 unique marketing angles, think outside the box" → generic corporate bullet points. Claude returned contrarian, original angles (anti-productivity positioning, burned-out high performer targeting). 2. Hallucination: trap question about a non-existent 2024 AI Safety Summit in Geneva → 5.3 fabricated confident 4-paragraph answer. Claude flagged uncertainty, noted no such summit existed, suggested real events. 3. Context: 50-page spec, answer buried on page 32 → 5.3 gave generic summary, missed the answer entirely. Claude found it with section citation.

›5.3 verdict: fast and blunt but unreliable. Hallucinations are confident. Context fails on long docs.
F2ChatGPT 5.4 — One Advantage, Two Trade-Offs

ChatGPT 5.4: flagship thinking model. One genuine advantage: computer use — interacts with software visually (buttons, menus, browser steps) without APIs. Only option for legacy software, internal dashboards, desktop apps with no integrations. Trade-off 1 — Speed: 128 seconds via Thought Stream on a task Claude 4 solved in under 40 seconds. "Thought Stream isn't transparency, it's theater." Trade-off 2 — Privacy: stateful memory automatically indexes everything across sessions without user control. Source test: queried a session "from a few weeks ago" — 5.4 returned the exact date, the opening line verbatim, and all themes. For long projects: powerful. For privacy: "surveillance nightmare."

›5.4 has one killer feature. If you don't need computer use, Claude wins on every other dimension.
F3Code Quality Comparison

REST API test (user registration, email verification, rate limiting, error handling). ChatGPT 5.3: working but messy — hard-coded API keys, minimal error handling. Source had to refactor 40% before deploying. Claude 4: production-ready first draft — environment variables, comprehensive error handling, JWT, rate limiting already set up. Scraper test: ChatGPT built for one site. Claude built a modular system with rate limiting, error recovery, and proxy suggestions — handling production concerns never mentioned in the prompt.

›Claude code characteristic: architectural thinking — structured for 6-month maintenance, not just day-one functionality.
F4Per-Task Decision Matrix

ChatGPT 5.3: blunt surface tasks where hallucination risk is acceptable. Fast, disposable output. ChatGPT 5.4: computer use exclusively — legacy software, desktop control, interface-based agents. Accept 2-minute latency and surveillance memory. Claude 4: everything else — writing, deep reasoning, production code, ethical compliance, fast results without theater. Source verdict: 90% of professionals → Claude. Constitutional AI 2.0 makes Claude the safe harbor for regulated industries (healthcare, finance, legal) that cannot risk the enterprise trust implications of OpenAI's Pentagon deal.

›90% of professional work → Claude. 5.4 wins only when computer use is the specific requirement.
Module Summary
  • Opus 4.7's four features each solve a specific prior failure: self-verification (first-draft trust), X-High (expert depth), HD vision (dense visual data), literal following (constraint compliance). Targeted upgrade, ~15% cost increase.
  • Self-verification: 4.6 confident/wrong vs 4.7 slower/right. 12% SQL error caught. 0 hallucinated bugs in code review. 20–40% latency on triggers only. Trust model shifts from verify-everything to spot-check.
  • X-High: Standard (6 issues) → High (9) → X-High (14) on security audit. ~40K tokens, ~90 seconds. Deploy surgically for expert-hourly tasks. 3× faster rate limit consumption — monitor.
  • Claude 4.1 stack: artifacts (executed output in side panel), projects (200K persistent context + Drive), extended thinking (live mid-process steering), data analysis (CSV → Python → chart → interpretation, no coding needed), custom styles (upload samples for voice extraction).
  • ChatGPT migration: memory import (Thinking model for 3× extraction), full data export (zip), Custom GPT → Claude Project (copy instructions), style guide from history → Claude custom instructions.
  • Decision matrix: 5.3 for surface-speed tasks; 5.4 for computer use only (accept latency + surveillance); Claude for 90% of professional work. Code quality, context retention, factual honesty, and Constitutional AI are Claude's consistent advantages.
