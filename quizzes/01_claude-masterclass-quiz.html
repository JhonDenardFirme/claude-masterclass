AAdaptive Thinking is a free-tier feature; Extended Thinking requires a Pro plan.
BExtended Thinking is a UI toggle that switches on deep reasoning for all queries in a session; Adaptive Thinking is triggered by specific phrases like "think carefully" or "ultra think" in the prompt itself.
CAdaptive Thinking uses web search to supplement reasoning; Extended Thinking operates purely on the model's internal knowledge.
DThey are functionally identical — Adaptive Thinking is simply the rebranded name for Extended Thinking in newer model versions.
6MC
A freelancer creates three separate Claude Projects: one for client emails, one for blog drafts, and one for code reviews. Which project management practice does this correctly implement, and why?
AIt is an inefficient approach — a single "Work" project with all tasks in one place would consolidate context and improve Claude's output quality.
BThe one-project-per-workflow-type rule — separate projects prevent context pollution and keep each workflow's instructions, examples, and constraints focused on a single purpose.
CIt is an acceptable approach, but only if all three projects share the same instruction set so Claude's tone remains consistent across task types.
DProjects are only useful for code-related tasks; email and blog drafts should be handled in regular chats without project context.
7MC
In Claude Code, why do negative constraints in claude.md ("don't touch schema.prisma", "don't use default exports") outperform equivalent positive instructions in improving first-attempt accuracy?
ANegative constraints are processed by a separate, higher-priority parsing layer in the Claude Code engine that positive instructions don't reach.
BNegative constraints set hard boundaries on what not to do, preventing the most common error patterns before they occur; positive instructions describe desired behavior that Claude might interpret ambiguously in context.
Cclaude.md only supports negative-constraint syntax; positive instructions written in that file are silently ignored by Claude Code.
DPositive instructions cause Claude to over-generate and apply changes too broadly; negative constraints produce fewer but higher-quality outputs.

BConnectors (MCP integrations) configured in Settings → Connectors, enabling Claude to read and act within multiple services simultaneously — with explicit user confirmation before any action.
CArtifacts with persistent storage, which can pull data from external services once published and connected to live data sources.
DThe Memory feature, which indexes all connected accounts automatically when enabled and makes that data available across sessions.

DCowork tasks are available on free plans; Claude Code Routines require an API billing account with a minimum spend threshold.

AClaude Code searches the web for the component library matching the screenshot's visual style and applies the matching fix to all files in the project.

12MC
The 1M token context window achieves 76% retrieval accuracy at context edges. Why is this metric specifically significant compared to prior models?
AIt confirms that Claude's response time remains constant regardless of how full the context window is, which enables faster real-time applications.
BPrior models showed only 18% accuracy at context edges, meaning information at the start of long documents was largely inaccessible; the improvement makes long-document analysis practically reliable.
CIt means users no longer need to upload multiple files; the 76% rate proves the model can reconstruct missing information from partial context alone.
DThe metric confirms that context compaction never loses critical information, making it safe to run compaction at any point during a session.
13MC
When should a developer trigger /compact during a Claude Code session?
AOnly at the start of a new session to load the previous session's context into the current window.
BMid-session when Claude starts giving inconsistent or looping answers, which indicates the context window is filling and noise is degrading output quality.
CAfter every five code edits to ensure that test results and error messages don't accumulate in the context window.
D/compact is automatically triggered by Claude Code when the context window reaches 80% capacity and does not need to be run manually.
14MC
A content creator uploads three previous articles to generate a Custom Style. What does this Custom Style specifically control, and how does it differ from Custom Instructions?
ACustom Styles control what Claude knows about the user (professional background, preferences); Custom Instructions control how Claude formats output.
BCustom Styles control how Claude writes — tone, voice, and phrasing patterns; Custom Instructions control what Claude knows about the user. Both can be active simultaneously.
CCustom Styles replace Custom Instructions when active; only one layer can be in effect at a time per session.
DCustom Styles are session-scoped and reset on every new chat; Custom Instructions are the only persistent user preference mechanism.

16MC
Why is Cowork's parallel sub-agent architecture specifically significant for multi-file document processing tasks, compared to running the same task in Chat?
ACowork can access cloud storage directly while Chat cannot, enabling larger file transfers for document-heavy workflows.
BCowork decomposes tasks into parallel sub-workflows — reading and writing multiple files simultaneously — producing finished native files in a fraction of the time that sequential Chat processing would require.
CChat processes files one at a time due to context window limits, while Cowork compresses each file before processing so more can fit in a single pass.
DBoth Chat and Cowork process files identically; the only difference is that Cowork saves outputs to the designated output/ folder automatically.

Opus 4.7's image resolution increase to 2,576 pixels unlocks which category of use case that was previously unreliable?
AGenerating high-resolution hero images for websites and marketing materials directly from prompts.
BReading fine text on document screenshots, parsing dense financial spreadsheets, and analyzing architectural drawings where prior lower resolution made text illegible.
CVideo frame analysis, since the higher pixel density allows individual video frames to be processed without quality loss during extraction.
DReal-time screen sharing, where Claude monitors a live desktop session and provides instant feedback on UI changes.
18MC
A project's claude.md file has grown to 450 lines covering both global preferences and project-specific rules. What is the recommended fix, and why?
ASplit the file into multiple claude.md files placed in subdirectories, which Claude Code automatically merges at session start.
BMove global rules (preferences applying to all projects) to ~/.claude/claude.md and keep only project-specific rules in the project's claude.md, trimmed to under 200 lines.
CCompress the file using /compact before each session to reduce its line count; Claude Code processes compressed files more efficiently.
DLength doesn't matter for claude.md — Claude Code always reads the full file regardless of size without any performance penalty.
19MC
Meta-prompting instructs Claude to "explain 3 things you changed" after rewriting a prompt. What is the primary pedagogical value of requesting this explanation?
AIt forces Claude to self-verify the rewritten prompt's accuracy against the original, reducing hallucinations in the output.
BThe explanation teaches the user what variables — role specificity, XML structure, success criteria, thinking triggers — most improve prompt quality, making them a better prompt writer over time.
CRequesting an explanation causes Claude to take more time before generating, which increases the model's effective "thinking budget" for the rewrite task.
DIt provides a version history that the user can submit to Anthropic to improve the prompt engineering guidelines in future model releases.

BThe session context is poisoned by accumulated failed attempts; start a fresh session with a clean description of the problem and only the minimally relevant code.
CSwitch to Opus mid-session — the repeated failure indicates that Sonnet lacks the reasoning capacity for this fix and a model upgrade is required.
