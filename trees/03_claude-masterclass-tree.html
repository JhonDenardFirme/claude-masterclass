Module Summary
  • Opus 4.7's four features each solve a specific prior failure: self-verification (first-draft trust, 12% SQL error caught, 0 hallucinated bugs), X-High thinking (6→14 security issues across tiers), HD vision (1,092px→2,576px), literal instruction following (constraints followed as hard limits).
  • Self-verification fires on code/logic/math/multi-step only. 20–40% latency on triggers = the accuracy dividend. Trust model shifts to "competent collaborator I spot-check." Use 4.7 precisely where correctness outweighs speed.
  • X-High: ~30K thinking tokens, ~40K total tokens, ~90 seconds. Surgical deployment only — expert-hourly tasks. Monitor rate limits (3× faster depletion). PDF vision workaround: screenshot page, upload image.
  • Claude 4.1 stack: artifacts (side-panel executed outputs, iterative surgical refinement), projects (200K permanent context, Drive integration), extended thinking (live mid-process steering — redirect without restart), data analysis (CSV→Python→chart→narrative), custom styles (upload samples for voice extraction).
  • Migration workflow in order: memory import (enable 3 personalization options, switch to Thinking model for 3× extraction), full data export (zip), style guide from history, Custom GPT → Claude Project (copy Instructions).
  • Decision matrix: 5.3 for surface speed tasks; 5.4 for computer use only; Claude for 90% of professional work. Code architectural quality, context retention (Redis/60s TTL found on page 32), factual honesty, and Constitutional AI are Claude's consistent edges.
